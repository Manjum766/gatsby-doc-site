PR Review Agent — Architecture & Design Plan

Automated Pull Request governance for IBM AEM as a Cloud Service (AEMaaCS) — ibm-cms-cloud

Repository: ibm-cms-cloud
Platform: GitHub Actions + watsonx / GPT-4o
Jira Project: ADCMS
Branch: develop

0 Overview

The PR Review Agent is a fully automated workflow triggered on every GitHub pull_request event (opened, edited, synchronize). It enforces enterprise PR standards, performs LLM-assisted code quality review, scans for security vulnerabilities, validates AEM cloud-readiness, and posts structured inline comments with suggested fixes — blocking merges on critical failures.

1 Six Core Capability Pillars

PR Template Compliance Pillar 1

Validates PR title format, Jira URL, ticket key consistency, and all required sections against the official PR template.

Code Quality Review Pillar 2

LLM-assisted static analysis on changed files — Java, JS/TS, SCSS, and HTL/Sightly with inline line-level comments.

Security & Vulnerability Scan Pillar 3

OWASP CVE checks on Maven & npm dependencies, Semgrep SAST, and secret/credential leak detection.

AEM Cloud Readiness Pillar 4

Enforces AEMaaCS-specific rules: mutable content placement, OSGi config naming, deprecated API detection, CODEOWNERS enforcement.

Dependency & Build Health Pillar 5

Detects pom.xml version drift, SNAPSHOT versions on release branches, bundle size regressions, and licence compliance.

AI-Suggested Fixes Pillar 6

For every issue found, the agent proposes a concrete, ready-to-apply code fix as an inline PR comment suggestion.

5 Pillar 4 — AEM Cloud Readiness

Cloud-readiness rules specific to AEMaaCS (ibm.com migration to AEMaaCS):

  • No mutable content in ui.apps/ — /content and /conf overlays must live in ui.content/
  • OSGi config naming in ui.config/ must follow <pid>~<runmode>.cfg.json pattern
  • No deprecated AEM APIs — flags com.day.cq.* where com.adobe.cq.* replacements exist
  • Replication API flagged as cloud-incompatible; Distribution API must be used instead
  • No /etc/clientlibs — must use /apps/.../clientlibs
  • CODEOWNERS enforcement — changes to ui.frontend.idl/js/main/ or ui.frontend.idl/scss/ require review from @Lucio-Poveda
  • analyse module reminder — links developer to run mvn clean install (triggers aemanalyser-maven-plugin)

9 Jira Integration

Because every PR title carries an ADCMS-XXXX key and the base URL is https://jsw.ibm.com/browse/, the agent performs automatic Jira lifecycle management:

TriggerJira Action
PR OpenedTransition ticket → In Review; post PR link as comment on Jira ticket
PR MergedTransition ticket → Ready for QA
PR Closed (no merge)Transition ticket → In Progress (revert)
Template ComplianceValidate ticket is not in Closed/Done state before allowing PR open
Acceptance CriteriaPull Jira description and cross-reference against PR 'Expected Results' section
